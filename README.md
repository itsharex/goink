写长篇小说最大的难题不是"写不出来"，而是"写了后面忘了前面"——角色性格在第五十章变了样，时间线在第三十章对不上，埋下的伏笔到结尾也没回收。市面上的 AI 写作工具大多是"你说一句它写一段"，关掉对话窗口一切归零，下一次又得重新交待。

Goink 不一样。它是一个桌面应用，双击打开就能用。AI 不是来替你写的，是来帮你记住和管理这个越来越复杂的故事世界的。对话驱动设计，告诉AI你的想法，全自动帮你写出来。不同于流水线的设计，Goink使得你能够轻松与AI协作。

你做过一次的决定——"主角的性格是热血正义""第五章要埋一个关于玉佩的伏笔""以后对话保持冷峻风格"——不会在下一次对话中被冲走。系统记住了，并且主动维护。

## 一切创作决策都被追踪

### 说过一次，永远生效

写了几十万字之后，你还记得第三十七章跟AI说"以后的角色对话保持冷峻"吗？系统记得。这就是**创作偏好**——你在对话中说过的任何长期规则，AI 会主动记录。全局偏好对所有小说生效，单书偏好只对当前作品起效。说一次就够，不需要反复强调。

### 角色：不仅仅是名字和简介

每个角色有详细的设定档案（性格、能力、背景），由 AI 根据你的创作持续更新。更重要的是**角色关系**——张三对李四是"师徒但暗中互相提防"，和李四对张三的"敬重但有所隐瞒"不是一回事。关系发生变化时，旧记录不会删除，而是保留为历史。你可以随时回顾一段关系是怎么演变的。

### 时间线：知道故事走到哪了

三档章节计划管理你的创作节奏：下一章具体写什么、近期几章的方向、远期的大致规划。

更重要的是**伏笔系统**。每一条伏笔都记录了目标回收章节和重要程度。快到回收点了系统会提醒，过了回收点还没回收会标记异常。AI 写新章节时会自动查看附近的伏笔状态，不会让你的伏笔石沉大海。

### 弧线：跨章节的叙事线索

一条"复仇弧"可能跨越第 5 章到第 50 章。弧线不是一句简单的备注，而是一串节点链（发现真相→接近仇人→对决→结局），每个节点关联目标章节。写完一章，AI 推进节点进度、校准后续目标。

通常情况下一个故事有 3–5 条并行弧线，系统帮助你在这么多线索中不迷失方向。

### 世界观：故事发生在什么样的世界

地点不只是名字列表。系统追踪两件事：层级包含（王国→王宫→大殿）和空间连通（A 和 B 之间由山路连通）。AI 可以查询单个地点的详情、子地点和连通关系，也可以查看完整的世界地图。

### 读者认知：你知道的，读者不一定知道

这是最"作家向"的模块。它在追踪三件事：读者已经知道了什么、读者在等什么答案、读者误解了什么。每条记录在哪章种下的、在哪章回收的。对于需要制造悬念和反转的作品来说，这个功能让你精确控制信息的释放节奏。

## 状态维护是核心，三重保障确保它不遗漏

很多 AI 工具的问题不是"做不到"，而是"做几次就不做了"。Goink 用三层机制确保状态维护不会被跳过：

**第一层：系统提示词写死在 Agent 的指令里。** "创作完成后立即进行状态维护。不是可选步骤。宁可多调几次工具把状态对齐，不要图省事跳过。"

**第二层：动态注入的维护提醒。** AI 完成一篇长篇正文替换后，系统会自动注入一条消息，列明需要执行的具体维护项目——检查角色变化、更新伏笔状态、推进弧线节点、刷新读者认知。不是笼统地说"去维护"，而是告诉你具体该检查什么。

**第三层：Review Agent 独立审读。** 创作完成后，Review Agent 会从头检查章节内容与系统状态的一致性。角色性格有没有前后矛盾？该回收的伏笔有没有遗漏？弧线节点需不需要校准？它像一位严格的编辑，找出 AI 自己没注意到的问题。

## 你的每一次确认

AI 不会直接修改你的正文。每次 AI 要改动章节内容，都会把差异展示给你看，等你点头再写入。你可以当场批准、拒绝、或者给反馈让 AI 修正。如果选自动模式，AI 可以连续多轮自由写作，编辑直接生效。

所有版本都在——任何一次 AI 的修改都可以撤回。你永远不会失去之前的版本。同时，你也可以回退AI的改动，正文与追踪的数据均会进行回退，回退到某次对话开始之前。（beta阶段暂不支持）

## 开箱即用

安装包里已经包含了写作所需的一切。不需要安装 Python、Node.js、数据库，也不需要 GPU。不需要像python应用，为了一个语义搜索，需要安装几个G的依赖。

下载 → 安装 → 打开 → 填入 API Key → 开始写。

支持 Windows、macOS、Linux。三平台开箱即用，安装包均小于60MB。

## 快速开始

从 [Releases](https://github.com/sigpanic/goink/releases) 下载对应平台的安装包：

- **Windows** — 运行安装程序
- **macOS** — 打开 DMG，拖入 Applications
- **Linux** — 运行 AppImage

需要自行准备 LLM API Key（内置 DeepSeek 模板，也兼容任何 OpenAI 格式的 API）。

Windows 下载安装包后 SmartScreen 可能会弹出"Windows 保护了你的电脑"提示，这是因为安装包没有数字签名（代码签名证书需要每年付费，年费约
$300–500）。代码本身保证无任何恶意行为。                                                                  

点击"更多信息"→"仍要运行"即可继续安装。安装包是 CI 自动构建的，每次 Release 的 sha256 校验和都附在发布页面上，可以验证文件完整性。

### 从源码构建

```bash
# 系统依赖（Ubuntu/Debian）
sudo apt install libsqlite3-dev libgtk-3-dev libwebkit2gtk-4.1-dev gcc

# Go 1.25+
git clone https://github.com/sigpanic/goink
cd goink

make deps      # 下载运行时依赖
make build     # 生产构建
make dev       # 开发模式（热重载）
```

| 层 | 选型 |
|---|---|

| LLM 客户端 | 原生 net/http + SSE（OpenAI 兼容格式） |
| 前端 | React 19 + TypeScript + Tailwind CSS 4 |
| UI 组件 | shadcn/ui（Radix UI） |

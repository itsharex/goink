## 1. 系统目标

### 1.1 核心目标
构建一个能够利用外部AI API高效生成小说的系统，支持：
- **长期连载**: 持续生成新篇章，保持连贯性
- **多书管理**: 方便管理多本小说
- **可视化追踪**: 追踪每本小说的进度

### 1.2 系统特点
- 基于外部AI API（DeepSeek等）
- 多Agent协作架构
- 记忆管理和上下文检索
- 一致性检查和审核机制

## 2. 两层Agent架构

### 2.1 架构说明

```
┌─────────────────────────────────────────────────────────────────┐
│                     第一层：开发Agent（我们）                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │前端Agent │  │后端Agent │  │Review    │  │主调度    │        │
│  │(agent_1) │  │(agent_2) │  │Agent     │  │Agent     │        │
│  │          │  │          │  │(agent_3) │  │(agent_4) │        │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘        │
│                          ↓ 开发完成 ↓                            │
└─────────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│                   第二层：目标系统Agent（未来）                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │主控Agent │  │写作Agent │  │审核Agent │  │记忆Agent │        │
│  │          │  │          │  │          │  │          │        │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘        │
│                          ↓ 运行时 ↓                              │
│                     AI小说生成系统                                │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 第一层：开发Agent（当前）

| Agent ID | 角色名称 | 职责 |
|----------|----------|------|
| agent_1 | 前端开发Agent | 开发前端界面、UI组件 |
| agent_2 | 后端开发Agent | 开发后端API、业务逻辑 |
| agent_3 | Review Agent | 代码审查、执行Commit |
| agent_4 | 主调度Agent | 项目协调、进度监控 |

### 2.3 第二层：目标系统Agent（未来）

| Agent名称 | 职责 | 说明 |
|-----------|------|------|
| **主控Agent** | 调度、索引、章节管理 | 保存所有章节和索引，分配写作任务 |
| **写作Agent** | 具体章节编写 | 基于前文生成新章节，保证连贯性 |
| **审核Agent** | 章节审核、伏笔管理 | 维护挖坑和填坑记录，承前启后 |
| **记忆Agent** | 记忆检索、上下文构建 | RAG检索，提供写作上下文 |

## 3. 目标系统Agent详细设计

### 3.1 主控Agent

**核心职责**:
- 小说项目管理
- 章节索引维护
- 任务调度和分配
- 进度可视化

**内部数据结构**:
```json
{
  "novels": [
    {
      "id": "novel_001",
      "title": "小说标题",
      "chapters": [
        {"id": "ch_001", "title": "第一章", "status": "completed"},
        {"id": "ch_002", "title": "第二章", "status": "writing"}
      ],
      "index": {
        "characters": ["角色1", "角色2"],
        "locations": ["地点1", "地点2"],
        "plot_threads": ["主线", "支线1"]
      }
    }
  ]
}
```

### 3.2 写作Agent

**核心职责**:
- 接收章节写作任务
- 基于上下文生成内容
- 保证前后连贯性
- 请求审核

**工作流程**:
```
接收写作任务
    ↓
请求记忆Agent提供上下文
    ↓
生成章节内容
    ↓
提交审核Agent审核
    ↓
    ├─ 通过 → 提交给主控Agent
    └─ 需修改 → 重新生成
```

### 3.3 审核Agent

**核心职责**:
- 章节内容审核
- 伏笔管理（挖坑/填坑）
- 一致性检查
- 质量评估

**伏笔管理数据**:
```json
{
  "foreshadowing": [
    {
      "id": "fs_001",
      "type": "挖坑",
      "chapter": "ch_003",
      "content": "神秘人物出现",
      "status": "unresolved"
    },
    {
      "id": "fs_002",
      "type": "填坑",
      "chapter": "ch_010",
      "content": "神秘人物身份揭晓",
      "status": "resolved",
      "resolved_chapter": "ch_003"
    }
  ]
}
```

### 3.4 记忆Agent

**核心职责**:
- 章节内容向量化存储
- 上下文检索（RAG）
- 角色记忆管理
- 情节时间线维护

**检索策略**:
- 滑动窗口：最近N章
- 语义检索：相关情节
- 角色检索：角色相关内容
- 时间线检索：按时间顺序

## 7. 技术栈

### 7.1 后端
- **框架**: FastAPI
- **数据库**: MySQL 8.0
- **向量数据库**: ChromaDB
- **LLM集成**: LangChain + DeepSeek API
- **工作流**: LangGraph

### 7.2 前端
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI库**: Ant Design
- **状态管理**: Zustand
- **路由**: React Router

### 7.3 AI
- **主模型**: DeepSeek Chat (128K上下文)
- **嵌入模型**: text-embedding-ada-002 或本地模型
